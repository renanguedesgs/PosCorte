Como ter marceneiros sem cadastrar na mão

O caminho real para automatizar a captação de montadores, plugar APIs e deixar o sistema rodando: você coda, o ciclo gira, a comissão entra.
Documento técnico-estratégico · Gerado para Renan · Junho/2026 · baseado no código atual do repositório PosCorte

Você pediu para eu ser revolucionário e honesto ao mesmo tempo. Então começo pela verdade que vale mais que qualquer fantasia — e em seguida te entrego o plano de engenharia que chega o mais perto possível do seu sonho de “só codar e a grana entrar”.

⚠️ A verdade que te deixa rico (a fantasia te deixa pobre)

Não existe uma “API onde tem marceneiros” que você pluga e sai dinheiro. Isso não existe para ninguém — nem Uber, nem iFood, nem GetNinjas tiveram isso. O que existe é uma máquina de captação automatizada que você constrói uma vez e depois roda quase sozinha. A boa notícia: 90% disso é código, e o código é justamente a sua arma. Os 10% restantes (CNPJ + conta de pagamento) são uma tarde de burocracia que destrava todo o resto.

💰 A jogada que realmente fica rica

Seu dinheiro não vem de “ter marceneiros”. Vem da comissão (take rate) sobre cada montagem paga com escrow. O marceneiro é o insumo; o arquiteto pagante é o cliente; o seu lucro é a ponte automatizada entre os dois. Quem automatiza essa ponte e domina o canal de distribuição (balcão de MDF) fica rico. Quem só fica esperando uma API mágica, não.

1. As 3 únicas formas reais de “conectar uma API de marceneiros”

Existem exatamente três fontes de oferta de montadores — e seu código já tem o esqueleto para as três. A estratégia vencedora é usar as três em camadas, da mais barata para a mais cara.

1

Auto-cadastro (o marceneiro se cadastra sozinho)

O melhor e mais barato. Você cria um link/landing “Seja um montador PósCorte”, o profissional preenche, cai no banco com Verificado=false. Zero cadastro manual seu.

2

Captação automática (robô que acha e convida)

Um job lê fontes públicas (Google Places, Instagram, marketplaces), encontra montadores por cidade e dispara convite por WhatsApp/e-mail com o link do item 1. Você não sai caçando ninguém.

3

API parceira (a “API de marceneiros” de verdade)

Helpie, Gaba, Parafuzo, GetNinjas etc. Você integra via o ProvedorService que já existe e despacha a ordem; eles alocam o montador. Mais caro, mas é “plugar e usar”.

🚫 O que NÃO existe (não perca tempo procurando)

Uma API pública, grátis e nacional que liste marceneiros de planejados com disponibilidade em tempo real. Os marketplaces (GetNinjas/Parafuzo) não expõem catálogo aberto — eles recebem o pedido e alocam internamente. Por isso o item 1 + item 2 são a sua independência; o item 3 é o atalho pago.

3. O motor automático (a arquitetura da máquina)

Este é o desenho do sistema que faz o ciclo girar sem você no meio:

Tudo que está nas caixas pode ser código. O único ponto humano saudável é a verificação do montador antes do primeiro serviço (selo Verificado=true) — e mesmo isso vira semi-automático com documento + selfie.

4. Roadmap de código — em sprints (o que VOCÊ desenvolve)

Sprint 1 — Auto-cadastro do montador (mata 80% do cadastro manual)

1
Endpoint público POST /api/v1/marceneiros/auto-cadastro recebendo nome, telefone, cidade, especialidades. Salva com Verificado=false, Disponivel=false, OrigemExterna="autocadastro".
2
Página pública /Marceneiros/SejaParceiro (Razor) — formulário simples, mobile-first, com o mesmo capricho da landing.
3
Fila de aprovação no admin — reaproveita /Admin/Marceneiros, com botão “Verificar”. 1 clique = montador entra na alocação automática.

Resultado: você divulga 1 link. Os montadores se cadastram sozinhos. Você só aprova.

Sprint 2 — Robô de captação (você para de caçar pessoas)

1
BackgroundService (igual ao LiquidacaoBackgroundService que você já tem) que consulta a Google Places API por “montador de móveis planejados” + cidade.
2
Deduplicação por OrigemExterna="places:PLACE_ID" — o campo já existe exatamente para isso.
3
Disparo de convite automático (WhatsApp Cloud API ou e-mail) com o link do Sprint 1. Quem responder, se cadastra. Quem não, fica como lead frio.

Resultado: a oferta cresce sozinha enquanto você dorme. Custo marginal ~zero.

Sprint 3 — Notificações reais (o sistema fala pelos dois lados)

1
Trocar o stub do NotificacaoService por WhatsApp Cloud API (Meta) ou Twilio.
2
Gatilhos automáticos: montador alocado → avisa arquiteto + montador; vistoria pendente → cobra arquiteto; escrow liberado → avisa montador.

Resultado: zero mensagem manual no seu WhatsApp pessoal.

Sprint 4 — API parceira como rede de segurança

1
Quando AlocarParaProjetoAsync não achar montador próprio disponível, cai para o ProvedorService (Helpie/Gaba). O código já tem EstaConfigurado para esse fallback.
2
Você nunca deixa um arquiteto pagante sem montador — e isso protege sua reputação e sua receita.

5. Onde a grana entra de verdade (o modelo)

20%
take rate sobre cada montagem (reenquadrado como “seguro/escrow” na UI)
PIX
Asaas em escrow: segura o dinheiro, libera após vistoria
72h
liquidação automática (já existe LiquidacaoBackgroundService)

O dinheiro entra sozinho a cada ciclo concluído — mas só depois de existirem (a) CNPJ + Asaas ligados e (b) os dois lados do marketplace girando. Sem isso, é uma demo linda que não fatura. Esta é a parte honesta: código sem credencial de pagamento = R$ 0,00.

7. Seu plano de 30 dias (foco brutal)

Semana 1 — Destravar dinheiro

Abrir MEI/CNPJ, conta Asaas, ligar Asaas__Enabled=true. Deploy + domínio. Sem isso nada fatura.

Semana 2 — Auto-cadastro

Codar Sprint 1 (endpoint + página + aprovação). Divulgar o link “Seja montador” em grupos de marcenaria.

Semana 3 — Robô + notificação

Codar Sprint 2 e 3. Robô busca e convida montadores; notificações reais nos dois lados.

Semana 4 — Primeiros ciclos pagos

3 pilotos com take rate 0% → depoimento em vídeo. Depois liga os 20%. A grana começa a entrar a cada ciclo.
